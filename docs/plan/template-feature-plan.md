# Feature Plan Template

Use this template for new implementation plans in `docs/plan/`.

---

# Title

## Summary

- What is being changed?
- Why is it needed?
- What does success look like?

## Current State

- How the relevant part of the repo works today
- Important constraints or dependencies
- Existing risks, pain points, or coupling

## Proposed Changes

- Core implementation approach
- Main subsystems or areas affected
- Any public or project-level conventions added or changed
- Important edge cases or compatibility requirements

## Testing

- Linting, type checks, or static verification needed
- Unit, component, integration, or end-to-end coverage expectations
- Manual verification scenarios if relevant

## Risks / Assumptions

- Key assumptions the plan depends on
- Rollout or regression risks
- Defaults chosen so the implementer does not need to guess

## Acceptance Checklist

- [ ] Implementation approach is clear
- [ ] Affected areas are identified
- [ ] Testing expectations are explicit
- [ ] Risks and assumptions are documented
- [ ] Another engineer could implement this without inventing missing decisions
